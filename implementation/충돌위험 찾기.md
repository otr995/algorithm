# 프로그래머스 [PCCP 기출문제] 3번 / 충돌위험 찾기

---

## 🧠 1. 첫 접근 (아이디어)
> 문제를 처음 보고 떠올린 생각 / 직관

- 1. BFS를 사용해서 최단 시간을 각각 로봇마다 구하자
### 착각한 부분
- 문제에서 최단 경로라는 말이 나와 관성적으로 BFS를 떠올림
### 문제점
- 충돌 횟수를 알려면 좌표를 알아야 함
- BFS로 시간만 구하면 이동 경로를 알 수 없음
- 문제에서 주어진 상황은 로봇이 어차피 최단 경로로 움직임
### 사고 전환
- 같은 시간에서 같은 좌표에 있는 로봇을 구해야함

- 2. 각 로봇마다 이동 경로를 기록하기
### 문제점
- 로봇의 수가 100대면 100*100 사이즈 좌표, 100개의 배열이 필요함
- 이동 경로를 모두 확인하면서 같은 시간에 같은 좌표를 확인해야 함
### 사고 전환
- map 자료구조를 사용하여 key,value 형태로 만들자

- 3. map과 배열 이용하기

## ❌ 2. 틀린 접근 / 시행착오

### 시도 1

[code]
// 틀린 코드
import java.util.*;
class Solution {
    public int solution(int[][] points, int[][] routes) {
        int answer = 0;
        int[][] pos = new int[routes.length][2];
        int[][] target = new int[routes.length][2];
        boolean[] done = new boolean[routes.length];
        
        for(int i=0; i<routes.length; i++){
            pos[i][0] = points[routes[i][0]-1][0];
            pos[i][1] = points[routes[i][0]-1][1];
            target[i][0] = points[routes[i][1]-1][0];
            target[i][1] = points[routes[i][1]-1][1];
        }
        
        while(true){
            Map<String,Integer> map = new HashMap<>();
            
            for(int i=0; i<routes.length; i++){
                if(done[i]) continue;
                
                String key = pos[i][0] + "," + pos[i][1];
                map.put(key,map.getOrDefault(key,0)+1);
            }
            
            for(int v : map.values()){
                if(v > 1) answer += 1;
            }
            
            for(int i=0; i<routes.length; i++){
                if(done[i]) continue;
                
                if(pos[i][0] != target[i][0]){
                    pos[i][0] += (pos[i][0] < target[i][0]) ? 1:-1;
                }
                else if(pos[i][1] != target[i][1]){
                    pos[i][1] += (pos[i][1] < target[i][1]) ? 1:-1;
                }
                else{
                    done[i] = true;
                }
            }
            
            boolean move = true;
            for(int i=0; i<routes.length; i++){
                if(!done[i]) move = false;
            }
            
            if(move) break;
            
        }
        
        return answer;
    } 
}
[/code]

- ❗ 왜 틀렸는지:
    - 목적지가 여러 개 (경유지가 있을 경우)를 고려하지 않음

class Solution {
    public int solution(int[][] points, int[][] routes) {

        int[][] pos = new int[routes.length][2];

                map.put(key,map.getOrDefault(key,0)+1);
            }
            
            for(int v : map.values()){
                if(v > 1) answer += 1;
            }
            
            for(int i=0; i<routes.length; i++){
                if(done[i]) continue;
                
                if(pos[i][0] != target[i][0]){
                    pos[i][0] += (pos[i][0] < target[i][0]) ? 1:-1;
                }
                else if(pos[i][1] != target[i][1]){
                    pos[i][1] += (pos[i][1] < target[i][1]) ? 1:-1;
                }
                else if(desidx[i] < routes[0].length -1){
                    desidx[i]++;
                    target[i][0] = points[routes[i][desidx[i]]-1][0];
                    target[i][1] = points[routes[i][desidx[i]]-1][1];
                }
                else{
                    done[i] = true;
                }
            }
            
            boolean move = true;
            for(int i=0; i<routes.length; i++){
                if(!done[i]) move = false;
            }
            
            if(move) break;
            
        }
        
        return answer;
    } 
}

- ❗ 왜 틀렸는지 :
    - 로봇을 이동하는 과정에서 경유지 도착했을 때 경유지 재설정 후 다음 루프에서 이동을 함
    - 경유지 재설정 후 이동하지 않아 한 루프(시간)가 느려짐 -> 동기화 실패

## 🔍 3. 핵심 관찰 (문제 제대로 이해하기)

- 중요한 조건:
    - 로봇은 routes[i]에 적힌 포인트를 순서대로 방문함
    - 이동은 항상 r 좌표 먼저, 그 후 c좌표 순서로 함
    - 같은 시간, 같은 좌표에 로봇이 2대 이상 있을 경우 충돌 1회
    - 도착이 끝난 로봇은 충돌 계산에서 제외
- 놓쳤던 부분:
    - routes가 2개 이상일 수 있음 (여러 포인트를 방문하는 경유지가 존재할 수 있음)
    - 각 로봇마다 시간을 재는 것이 아니라 모든 로봇을 같은 시간으로 동기화 시켜야함 
    - 각 로봇마다 현재 향하는 목적지 인덱스를 따로 관리해야 함
    - 경유지에 도착했을 때, 목적지 재설정 후 멈추는게 아니라 바로 이동해야 함
- 핵심 포인트:
    - 시간 단위로 모든 로봇을 동기화
    - 충돌 체크는 이동 전 현재 위치 기준
    - map을 루프마다 초기화 시켜 key 값에 좌표를 넣어 동일 좌표 충돌 여부를 체크함
    - 루프 종료 기준을 각 로봇 도착 여부로 판별

## ✅ 4. 최종 접근 방법

- 전체 전략:
    - 모든 로봇을 동시에 1초씩 시뮬레이션하면서, 매 시간마다 같은 좌표에 있는 로봇 수를 센다
- 사용한 자료구조:
    - int[][] pos: 현재 로봇 위치
    - int[] desidx: 각 로봇이 현재 향하는 목적지 인덱스
    - boolean[] done: 이동 완료 여부
    - Map<String, Integer>: 시간별 좌표 중복 개수 저장
- 로직 흐름:
    1. 각 로봇의 시작 위치를 routes[i][0]으로 초기화한다.
    2. 매 시간마다 완료되지 않은 로봇들의 현재 좌표를 Map에 기록한다.
    3. 같은 좌표에 2대 이상 있으면 충돌 위험을 1 증가시킨다.
    4. 각 로봇이 현재 목적지에 도착했는지 확인한다.
    5. 도착했다면 다음 목적지로 갱신하거나, 마지막 목적지면 완료 처리한다.
    6. 완료되지 않은 로봇은 r 좌표 우선, c 좌표 후순위로 한 칸 이동한다.
    7. 모든 로봇이 완료되면 반복을 종료한다.

## 💻 5. 최종 코드

[code]
import java.util.*;
class Solution {
    public int solution(int[][] points, int[][] routes) {
        int answer = 0;
        int[][] pos = new int[routes.length][2];
        int[][] target = new int[routes.length][2];
        boolean[] done = new boolean[routes.length];
        int[] desidx = new int[routes.length];
        Arrays.fill(desidx,1);
        
        for(int i=0; i<routes.length; i++){
            pos[i][0] = points[routes[i][0]-1][0];
            pos[i][1] = points[routes[i][0]-1][1];
            target[i][0] = points[routes[i][desidx[i]]-1][0];
            target[i][1] = points[routes[i][desidx[i]]-1][1];
        }
        
        while(true){
            Map<String,Integer> map = new HashMap<>();
            
            for(int i=0; i<routes.length; i++){
                if(done[i]) continue;
                
                String key = pos[i][0] + "," + pos[i][1];
                map.put(key,map.getOrDefault(key,0)+1);
            }
            
            for(int v : map.values()){
                if(v > 1) answer += 1;
            }
            
            for(int i=0; i<routes.length; i++){
                if(done[i]) continue;
                
                if(pos[i][0]==target[i][0] && pos[i][1]==target[i][1]){
                    if(desidx[i] < routes[0].length -1){
                        desidx[i]++;
                        target[i][0] = points[routes[i][desidx[i]]-1][0];
                        target[i][1] = points[routes[i][desidx[i]]-1][1];
                    }
                }
                if(pos[i][0] != target[i][0]){
                    pos[i][0] += (pos[i][0] < target[i][0]) ? 1:-1;
                }
                else if(pos[i][1] != target[i][1]){
                    pos[i][1] += (pos[i][1] < target[i][1]) ? 1:-1;
                }
                else{
                    done[i] = true;
                }
            }
            
            boolean move = true;
            for(int i=0; i<routes.length; i++){
                if(!done[i]) move = false;
            }
            
            if(move) break;
            
        }
        
        return answer;
    } 
}

## ⏱ 6. 시간 / 공간 복잡도

- 시간 복잡도: O(T × R)
    R: 로봇 수
    T: 모든 로봇이 이동을 마칠 때까지 걸리는 총 시간
- 공간 복잡도: O(R)
    로봇 위치, 목적지 인덱스, 완료 여부, 현재 시간의 좌표 Map을 저장한다.
---

## 😵 7. 어려웠던 점
- map을 사용해서 같은 시간 + 같은 좌표를 계산하는 아이디어가 떠오르지 않았다.
- routes[i]의 경유지를 끝까지 따라가야 하는데 처음에는 두 지점만 이동하도록 구현했다.
- 목적지 인덱스를 모든 로봇이 공유하는 변수로 둬서, 로봇별 진행 상태가 꼬일 수 있었다.
- 경유지 도착 후 다음 목적지로 넘어가는 타이밍을 잘못 잡으면 로봇이 한 칸 멈춘 것처럼 계산된다.

## 📚 8. 배운 점 (핵심 정리)
- 여러 객체가 동시에 움직이는 문제는 시간 단위 시뮬레이션으로 생각하면 정리하기 쉽다.
- 각 객체마다 상태가 다르면 반드시 배열로 따로 관리해야 한다.
- 충돌 문제에서는 언제 위치를 체크하는지가 중요하다.
- 시작 시점도 충돌 위험 계산에 포함될 수 있다.
